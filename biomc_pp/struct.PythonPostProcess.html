A struct that wraps the PostProcess type for Python bindings.

The PythonPostProcess struct is designed to provide a Python interface for the PostProcess struct, enabling the interaction with Rustโs PostProcess type in Python.

ยงFields

  • inner: A reference to the inner PostProcess object, which holds the actual post-processing logic that can be used in Python.

ยงExample

# Example usage in Python

pub(crate) fn new(folder: &str, root: Option<String>) -> PyResult<Self>

Creates a new instance of PythonPostProcess.

This function serves as a constructor for creating aPostProcess struct for use in Python. It takes a folder path and an optional root string as arguments to initialize the PostProcess object.

ยงArguments
  • folder (str): A path to the folder where the post-processing files or resources are located.
  • root (Option<String>): An optional root path used for additional processing logic, or None if not provided.
ยงReturns
  • PyResult<Self>: On success, returns a PythonPostProcess instance wrapped in a PyResult. On failure, raises a PyValueError with an error message.
ยงExample
post_process = PostProcess("path/to/folder", "optional/root")
ยงErrors

If the PostProcess::new function fails (e.g., due to invalid paths or other internal errors), this function returns a PyValueError with the message "Error creating object".
